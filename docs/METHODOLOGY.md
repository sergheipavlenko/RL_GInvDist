## 1. Постановка задачи как марковского процесса принятия решений

### 1.1 Пространство состояний S

Состояние описывается 8-мерным вектором структурных признаков полиномиальной системы:

| Индекс | Признак              | Формула нормализации             |
|--------|----------------------|----------------------------------|
| 0      | n_vars_norm          | n_vars / max_vars                |
| 1      | max_degree_norm      | max_deg / 20                     |
| 2      | avg_degree_norm      | avg_deg / 20                     |
| 3      | n_polys_norm         | n_polys / 50                     |
| 4      | avg_monomials_norm   | avg_mon / 200                    |
| 5      | density              | ∈ [0, 1]                         |
| 6      | sparsity             | 1 − density                      |
| 7      | complexity_index     | min(n_polys² · (max_deg+1)/1000, 10) / 10 |

### 1.2 Пространство действий A

```
A = {0: LEX, 1: GREVLEX, 2: REVGREVLEX}
```

- **LEX** — лексикографический порядок. Наглядный, но вычислительно дорогой.
- **GREVLEX** — градуированный обратный лексикографический. Оптимален в большинстве случаев.
- **REVGREVLEX** — степенно-обратно-лексикографический. Промежуточная производительность.

### 1.3 Функция вознаграждения R

```
R(s, a) = (t_LEX − t_chosen) / t_LEX
```

Интерпретация:
- R = 0 при выборе LEX (базовый уровень)
- R > 0 при выборе порядка быстрее LEX
- R < 0 при выборе порядка медленнее LEX

### 1.4 Гипотезы

**H₀:** Выбор мономиального порядка не влияет на вычислительную сложность статистически значимо, или RL-агент не способен обучиться полезной политике.

**H₁:** Агент, обученный методом PPO, способен предсказывать оптимальный порядок с точностью, обеспечивающей среднее ускорение ≥ 1.20x по сравнению с фиксированным LEX.

**Результат:** H₁ подтверждена. Достигнуто среднее ускорение **1.86x**, точность выбора **80.4%**.

## 2. Алгоритм PPO

### 2.1 Clipped Surrogate Loss

```
L_CLIP(θ) = E_t [ min(r_t(θ) Â_t, clip(r_t(θ), 1−ε, 1+ε) Â_t) ]
```

где `r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)` — отношение вероятностей.

### 2.2 Generalized Advantage Estimation (GAE)

```
Â_t = Σ_{l=0}^{∞} (γλ)^l δ_{t+l}
δ_t = r_t + γ V(s_{t+1}) − V(s_t)
```

### 2.3 Полная функция потерь

```
L(θ) = L_CLIP(θ) − c₁ · L_VF(θ) + c₂ · S[π_θ](s_t)
```

- `L_VF` = MSE между предсказанной и фактической ценностью
- `S` = энтропийный бонус для исследования

### 2.4 Гиперпараметры

| Параметр        | Значение | Обоснование                            |
|-----------------|----------|----------------------------------------|
| learning_rate   | 3×10⁻⁴   | Стандарт для PPO (Schulman et al.)     |
| γ (gamma)       | 0.99     | Малые эпизоды, долгосрочная награда    |
| λ (GAE lambda)  | 0.95     | Баланс bias-variance в GAE             |
| ε (clip)        | 0.2      | Стандарт PPO                           |
| entropy_coef    | 0.01     | Поощрение исследования                 |
| value_coef      | 0.5      | Стандартный вес value loss             |
| batch_size      | 64       | Достаточен для 8 систем × 3 порядка   |
| n_epochs        | 4        | Повторное использование опыта          |
| grad_clip       | 0.5      | Предотвращение взрыва градиентов       |

## 3. Архитектура нейронной сети

```
Входной слой: 8 признаков
        │
   ┌────┴────┐
   │         │
Policy     Value
Network    Network
   │         │
Lin(8→128) Lin(8→128)
ReLU       ReLU
Lin(128→64) Lin(128→64)
ReLU       ReLU
Lin(64→3)  Lin(64→1)
   │         │
Logits(3)  Value(1)

Итого: 19 076 параметров (с bias-членами)
Инференс: < 1 мс на CPU
```

**Обоснование выбора MLP:**
- Входной вектор признаков не имеет пространственной/временной структуры → CNN/RNN избыточны
- Малый размер модели важен для быстрого инференса (<< времени вычисления базиса)
- Архитектура Actor-Critic позволяет одновременно обучать политику и функцию ценности

## 4. Стратегия обучения и валидации

### 4.1 Разбивка данных

```
Обучение:   8 бенчмарковых систем × 100 эпизодов
Валидация:  все 8 систем (скользящая оценка каждые 10 эпизодов)
Тест:       оценка точности предсказания оптимального порядка
```

### 4.2 Ограничения

1. **Размер пространства действий**: 3 фиксированных порядка. Расширение до всех n! перестановок переменных требует другой архитектуры (например, pointer network).
2. **Обобщаемость**: модель обучена на 8 системах; обобщение на системы с числом переменных > 8 не гарантировано.
3. **Имитационное окружение**: на начальном этапе использовались предвычисленные данные; после интеграции с GInvDist v2.1.0 агент переобучен на реальных данных.

## 6. Воспроизведение результатов

```bash
# 1. Установка зависимостей
pip install -r requirements.txt

# 2. Обучение агента (100 эпизодов)
python train.py --episodes 100 --seed 42

# 3. Только оценка обученной модели
python train.py --eval-only --save-dir models/

# 4. Генерация графиков и таблиц
python experiments.py

# 5. Анализ архитектуры + ONNX экспорт
python architecture_analysis.py

# 6. Запуск тестов
pytest tests/ -v
```
