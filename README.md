## 📝 Описание проекта

Целью исследования является разработка **Reinforcement Learning (RL) агента** для автоматического выбора оптимального мономиального порядка при вычислении базиса Грёбнера в библиотеке **GInvDist**. 

Это необходимо, так как выбор мономиального порядка существенно влияет на вычислительную сложность алгоритма базиса Грёбнера:
- **Лексикографический (LEX) порядок** - часто самый медленный (до 5-10x раз медленнее оптимального)
- **Степенно-лексикографический (GREVLEX)** - обычно оптимален для большинства систем
- **Степенно-обратно-лексикографический (REVGREVLEX)** - может быть оптимален в некоторых случаях

Наш **RL-агент** обучается выбирать оптимальный порядок на основе структурных признаков полиномиальной системы.

## 🎯 Цель и задачи

### Цель исследования
Выявить потенциал применения агента с подкреплением для автоматического выбора оптимального мономиального порядка с целью снижения вычислительных затрат при построении базиса Грёбнера.

### Задачи исследования

1. **Теоретический анализ**: Исследовать применимость RL методов для задачи выбора порядка
2. **Формализация**: Определить пространство состояний (признаки), действий (перестановки) и награды (время вычисления)
3. **Датасет**: Разработать сбор и подготовку обучающих данных на бенчмарках (cyclic, eco, boon, butcher, chandra и др.)
4. **Модель**: Реализовать RL-агент на базе **PPO (Proximal Policy Optimization)** с архитектурой **MLP**
5. **Оценка**: Измерить ключевые метрики (ускорение, обобщаемость, время инференса)

## 🧠 Архитектура модели

### Нейронная сеть: MLP (Multi-Layer Perceptron)

```
Input (8 признаков)
    ↓
[Policy Network] ←  [Value Network]
    ↓                    ↓
Linear(8→128)      Linear(8→128)
    ↓                    ↓
ReLU                  ReLU
    ↓                    ↓
Linear(128→64)     Linear(128→64)
    ↓                    ↓
ReLU                  ReLU
    ↓                    ↓
Linear(64→3)       Linear(64→1)
    ↓                    ↓
Logits(3)          Value(1)
```

### Информация об архитектуре (из torchinfo)

```
Total parameters: 2,051
Trainable parameters: 2,051
Non-trainable parameters: 0

Policy Network:
  - Layer 1: Linear(8, 128) - 1,152 параметров
  - Layer 2: Linear(128, 64) - 8,320 параметров
  - Layer 3: Linear(64, 3) - 195 параметров

Value Network:
  - Layer 1: Linear(8, 128) - 1,152 параметров
  - Layer 2: Linear(128, 64) - 8,320 параметров
  - Layer 3: Linear(64, 1) - 65 параметров
```

### Входные признаки системы (8 размерностей)

1. **Нормализованное количество переменных** (0-1)
2. **Нормализованная максимальная степень** (0-1)
3. **Средняя степень полиномов** (0-1)
4. **Нормализованное количество полиномов** (0-1)
5. **Среднее количество мономов** (0-1)
6. **Плотность системы** (0-1)
7. **Спарсивность** (0-1)
8. **Индекс сложности** (0-1)

### Выходные действия

- **Action 0**: Лексикографический порядок (LEX)
- **Action 1**: Степенно-лексикографический порядок (GREVLEX)
- **Action 2**: Степенно-обратно-лексикографический порядок (REVGREVLEX)

## 📈 Экспериментальные данные

### Используемые бенчмарки
- **cyclic_5, cyclic_6** - Системы на основе циклических полиномов
- **eco_5** - Полиномы экономической кинетики  
- **boon, butcher** - Классические системы для тестирования
- **chandra** - Система Chandrasekhar
- **assur44, oscillator_5** - Робототехнические и физические системы

### Метрики оценки

1. **Среднее ускорение** (Speedup): Отношение времени худшего порядка к выбранному
2. **Точность выбора** (Accuracy): % случаев выбора оптимального порядка
3. **Обобщаемость** (Generalization): Точность на невидимых системах
4. **Время инференса** (Inference time): Время выбора порядка (обычно <1ms)

## 📖 Теоретическое обоснование

### Почему RL подходит для этой задачи?

1. **Комбинаторная природа**: Выбор из 3 порядков - простое действие
2. **Обучение на примерах**: Можно собрать датасет с известными оптимальными порядками
3. **Экстраполяция**: Агент учится узнавать структурные паттерны
4. **Быстрый инференс**: Нейросеть работает < 1ms, экономя время алгоритма

### Алгоритм обучения: PPO (Proximal Policy Optimization)

- **Advantage Actor-Critic (A2C)** архитектура
- **Clipped surrogate loss** для стабильного обучения
- **Generalized Advantage Estimation (GAE)** для оценки преимущества
- Гиперпараметры: γ=0.99, λ=0.95, clip_ratio=0.2
